1) Explain the terms:
a. Hashing
It is the process of applying a hash function to the input data in order to produce a fixed-size hash value.
h(data) = hashVal

Detailed explanation:
  • It is an irreversible (not exactly) process of transforming data to fixed-size hash values for the verification of data integrity, indexing, and quick data retrieval.
    • it is the process of applying a hash function to the input data in order to produce a fixed-size hash value
  • Alternatively, (regardless of the size of the input data).
  • A hash function is a function that converts the input and generates a unique hash value of fixed length as output.
  • It is not exactly irreversible, but rather computationally infeasible to reverse engineer it.

Properties of a good hash function include
  • Collision resistance (minimizing the likelihood of different inputs having the same hash value)
  • Avalanche effect (a.k.a anti-clustering, small change in input data results in significant change in output hash value, so that the reverse engineers can't derive any meaningful information regarding the hash)
  • Deterministic (hash function is a function in the mathematical sense)
b. Target Hash
  • In PoW blockchains, each block has a unique identifier known as the block hash. This hash is calculated by taking the data in the block (including transactions, timestamp etc.) and running it through a cryptographic hash function (e.g. SHA-256).
  • Miners must find a random number and combine it with the block data in such a way that the resulting block hash must be less than or equal to some threshold value.
  • The random number is called the Nonce and the threshold value is called the Target Hash.
  • Finding a nonce that produces the Target Hash is the proof of work. It requires significant computational effort and energy. The first miner to find a valid Nonce gets to create the next block in the blockchain and is rewarded for their work.
c. NFT
  • A non-fungible token (NFT) is a unique digital identifier that is recorded on a blockchain and is used to certify ownership and authenticity of digital assets such as digital art, music, audio, videos, etc.
  • Artists and creators can tokenize their work as NFTs, enabling them to monetize digital content and retain ownership rights.
  • They are non-fungible because unlike cryptocurrencies such as Bitcoin or Ethereum, where each token is interchangeable and of equal value, NFTs are unique and cannot be exchanged on a one-to-one basis.
  • NFTs allow individuals to buy, sell, and trade ownership of digital assets in a decentralized manner.
  • There are online marketplaces dedicated to buying, selling, and trading NFTs (e.g. NBA Top Shot).
d. Consensus Protocol
  • In the context of communication systems, a Protocol is a set of rules and guidelines that govern how data is transmitted, received, and processed, which helps devices to effectively exchange information and interact with one another.
  • In a similar fashion, the consensus protocol is a set of rules and procedures that enable the nodes in a network to agree on the state of a shared ledger.
  • They primarily maintain the integrity of the blockchain technology and ensure its security implicitly in the protocol; In other words, different consensus protocols can lead to varying degrees of decentralization which impacts the security of the network.

Commonly Used Consensus Protocols:
  • PoW - Proof of Work involves miners competing to solve complex mathematical puzzles. The first miner to solve the puzzle gets to add a new block to the blockchain.
  • PoS - In Proof of Stake system, the validators are chosen to create new blocks based on the amount of cryptocurrency they are willing to stake as collateral. This encourages the validators to follow the protocol to protect their staked assets.
